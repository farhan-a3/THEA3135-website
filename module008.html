3. Description of the System

Facebook is a social media platform where users can create profiles, connect with friends, share posts, and follow other pages. To participate, users are asked to make an account through a single registration form that collects basic personal and login information. This sign-up form helps Facebook identify users, secure their accounts, and connect them to the social network. It also links the new account to various features such as the news feed, friend suggestions, and targeted content.

4. Data Collected

Visible form fields

  • First name
  • Last name
  • Mobile number or email address
  • New password
  • Date of birth
  • Gender

Likely inferred/background data during sign-up

  • IP address and approximate location
  • Device type (phone, tablet, desktop)
  • Operating system and browser type
  • Time and date of registration
  • Referral source (like link from an app, search engine, or ad)

Behavioural data likely tracked after sign-up

  • Profiles and pages you view, like, or follow
  • Posts you react to, comment on, or share
  • Friends added and groups joined
  • Time spent in the feed and which types of content you pause on
  • Devices and locations used to log in
  • Interactions with ads (views, clicks, etc)

5. Relational Database Sketch (Conceptual)

Primary table: users

Field Key / role Description
user_id PRIMARY KEY Unique ID for each user account.
first_name Attribute First name entered on the sign-up form.
last_name Attribute Last name entered on the sign-up form.
email_or_phone Attribute (unique) Email address or mobile number used for login and contact.
hashed_password Security Encrypted version of the user’s password.
date_of_birth Attribute Date of birth used for age checks and personalization.
gender Attribute Gender field chosen during sign-up.
created_at Metadata Date and time the account was created.
country Attribute (inferred) Country / region, often inferred from IP or user settings.

Related table: posts

Field Key / role Description
post_id PRIMARY KEY Unique ID for each post created on the platform.
user_id FOREIGN KEY Links each post back to the account in users.user_id.
content_text Attribute Text content written by the user.
media_url Attribute Link to any attached image, video, or other media.
created_at Metadata Date and time the post was created.
visibility Attribute Audience setting (for example: public, friends, custom).

The users table stores each Facebook account, and the posts table stores content created by those accounts. The two tables are connected by user_id, which lets the system link each post to the correct user, build profile timelines, and generate the news feed from related posts.

6. Ethical Analysis

One ethical concern is privacy and surveillance, since Facebook combines registration data with detailed behavioural tracking (friends, posts, reactions, and ad clicks) to build deep profiles of users' social lives and interests. Another concern is data minimization and long-term retention: Facebook can store years of personal content, messages, and metadata that go far beyond what is needed to maintain a basic account. If this information were mishandled or breached, it could expose sensitive details about relationships, locations, and beliefs. There is also a risk of opaque personalization and influence, where the news feed and ads are shaped by algorithms using this data in ways users cannot easily see or control.

7. Recommendations

  • Make it clearer during sign-up which fields are strictly required and why, and provide a minimal sign-up option that only uses essential data.
  • Offer a simple privacy dashboard that highlights key categories of stored data (friends, posts, ad interests, location history) and lets users delete or limit what is kept.
  • Explain in plain language how news feed ranking and ad targeting use user data, and allow users to easily reduce tracking or opt out of certain types of behavioural profiling.
